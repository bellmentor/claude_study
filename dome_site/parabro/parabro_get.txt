url : https://www.parabro.co.kr/

1. 로그인 : https://www.parabro.co.kr/member/login.php
   - ID  : input#loginId (name=loginId)
   - PW  : input#loginPwd (name=loginPwd)
   - 로그인 버튼 : button[type=submit] (텍스트 "로그인")

2. 주문조회 : https://www.parabro.co.kr/mypage/order_list.php
   - 조회 시작일 : input#picker2 (name=wDate[], class=js_datepicker)
   - 조회 종료일 : input[name="wDate[]"]:not([id]) (두 번째 wDate[])
   - 조회 버튼   : button.btn_date_check (텍스트 "조회")
   ★ 조회기간 주의: 해당월 딱 맞춰(예 3/1~3/31) 조회하면 파라브로 서버 오류로 주문목록을
     제대로 못 가져온다. 그래서 앞뒤로 넓혀 조회한다.
       - 조회 시작일 = (선택월 전달의 마지막 날) − 3일
       - 조회 종료일 = 선택월 다음달의 3일
       예) 3월 선택 → 2/25 ~ 4/3 (윤년/30·31일은 date 연산이 자동 처리)
     실제 월 필터링은 summarize 에서 한다.

3. 엑셀 다운로드 : span.js_order_excel (class="skinbtn point2 btn_review_write js_order_excel", 텍스트 "주문내역 엑셀다운로드")
   ★ 이 '엑셀' 은 확장자만 .xls/.xlsx 이고 실제 내용은 HTML 테이블이라 엑셀에서 바로 열면
     데이터가 깨져 보인다. 다운로드 직후 pd.read_html → to_excel 로 진짜 xlsx 로 변환 저장한다.

4. 매입금 계산 (summarize)
   - 먼저 '주문날짜' 가 선택 기간(예 6/1~6/30) 안인 행만 남긴다(넓게 조회했으므로).
   - '주문상태' 에 취소/반품/교환/환불 이 포함된 주문은 제외 (공용 dome_site/order_filters.py).
   - 주문항목 중 품목명에 '적립금' 이 포함된 행은 삭제(제외).
   - 배송비는 '송장번호 단위'로 1번만 더한다(같은 송장번호=배송 1건). 송장 빈 행은 각각 1건.
   - (상품금액 합계) + (3,000원 × 배송건수) = 해당 월의 매입금.
   - 검증: 2026년 4월 = 148,360원 (14건→기간11건→취소류3건 제외 8건, 상품 124,360 + 배송 24,000).
